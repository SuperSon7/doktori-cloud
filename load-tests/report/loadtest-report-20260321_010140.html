1. Meeting Search API P95: 4.69s

Root Cause: MeetingRepositoryImpl.searchMeetings() — JOIN Book subquery executed twice per request

Impact: Stress test에서 전체 에러율을 36%까지 끌어올림. DB 커넥션 풀 고갈 유발.

Evidence: search 단독 테스트 P95 4.69s vs load 전체 P95 30ms — 검색이 10x 이상 느림

Action: 서브쿼리 중복 제거 또는 검색 결과 캐싱 (Redis)

2. HPA Scale-out Limit Max 4 Pods

Root Cause: API HPA maxReplicas=4. 1500 VU에서 Pod 4개로는 부족.

Impact: CPU 60% 트리거 → 2→4 확장됐지만, 여전히 30% 에러

Action: maxReplicas 증가 (4→8) + 워커 노드 확인

3. WebSocket Chat UNTESTED

Root Cause: chat-websocket.js에서 ws.connect()가 성공하지 못함 (ws_connect_duration=0s)

Impact: Chat 서비스 부하가 실질적으로 검증되지 않음

Action: WebSocket 스크립트 디버깅 + ALB WebSocket 라우팅 검증

4. ECR Image Pull on Scale-out 403 Forbidden

Root Cause: etcd encryption 키 불일치 → Secret 읽기 실패 → imagePullSecrets 인증 불가

Impact: HPA가 Pod를 늘려도 새 Pod가 뜨지 못함 (ImagePullBackOff)

Action: etcd encryption 키 동기화 + kubelet credential provider 전환

## FI-4: CPU Stress (HPA 트리거 검증)

- 실행 시각: 2026-03-25 16:09:27
- 부하 상태: load 실행 중 (API CPU 이미 212%)
- 결과: ⚠️ 부분 검증 — CPU stress 주입 성공하지만 HPA 스케일아웃 불가 (이미 maxReplicas)

### 관측값

| 지표 | 기대 | 실측 |
|------|------|------|
| CPU stress 주입 | CPU 80% 추가 부하 | ✅ 212% → **500~536%** (기존 부하 + stress) |
| HPA 스케일아웃 | 2→3~4 | ❌ **이미 4(maxReplicas)** — 추가 스케일아웃 불가 |
| 서비스 영향 | latency 증가 | CPU 500%에서 서비스 상태 확인 필요 (Grafana) |
| stress 해제 후 | CPU 정상화 | 해제 직후 533% (기존 부하가 아직 높음) |

### 발견
1. **Chaos Mesh StressChaos는 정상 동작** — CPU를 500%+까지 올림 (pod-kill과 다르게 작동)
2. **HPA 검증 불가** — 부하 테스트로 이미 maxReplicas 도달 상태에서 CPU stress를 추가했으므로 HPA 스케일아웃을 관찰 불가
3. CPU 500%에서도 Pod가 OOMKill 안 됨 — API에 CPU limit이 없어서 throttle 대신 노드 CPU를 계속 소비
4. **재검증 필요**: 부하 없는 안정 상태(CPU 5~10%)에서 stress 주입 → HPA 2→3→4 스케일아웃 과정 관찰

### 후속 조치
- 부하 없는 상태에서 FI-4 재실행하여 HPA 스케일아웃 순수 검증
- 또는 maxReplicas를 임시로 6~8로 올려서 스케일아웃 여유 확보 후 재검증
